# lesson.md — Lessons Learned

> 이 문서는 **사용자가 "이건 고쳐라"라고 지적한 실패**와 **반복하면 안 되는 함정**을 기록하는 살아있는 문서입니다.
> 새 문제가 생기거나 같은 실수를 반복하면 항목을 추가하세요. 형식: 무엇이 잘못됐나 → 왜 → 어떻게 고쳤나 / 규칙.

## 작성 규칙
- **증상(What)**: 사용자가 본 잘못된 동작/결과.
- **원인(Why)**: 근본 원인.
- **해결/규칙(Fix)**: 다음에 같은 길로 안 빠지는 구체적 규칙. 코드 위치는 `index.html`의 섹션 번호로 표기.

---

## L1. PNG 저장에 화면 축소(transform scale)가 그대로 찍힘
- **What**: 캔버스를 `--view:0.5`로 축소 표시하는데, 그 상태로 html2canvas를 돌리면 결과 PNG가 작게/잘리게 캡처될 수 있음.
- **Why**: html2canvas는 일부 상황에서 CSS `transform`을 반영해 캡처 박스를 계산함.
- **Fix**: 저장 직전 `body.exporting`을 켜고 `.slide{transform:none!important}`, `.stagewrap`을 1080×1350로 강제. `savePng()`는 `width/height/windowWidth/windowHeight`를 1080×1350로 명시. (섹션 8) **새 요소를 추가하면 항상 export 경로에서 깨지지 않는지 확인.**

## L2. 떠 있는 편집 UI가 저장 이미지에 포함됨
- **What**: 안전영역 점선·블록 외곽선·툴바가 PNG에 찍힐 위험.
- **Why**: 편집 보조 요소가 캔버스 안/위에 그려짐.
- **Fix**: 저장 전 `clearSel()`+툴바 숨김, `body.exporting`에서 `.guides/.block-bar/.blk.sel:after` 숨김 처리. (섹션 8 + CSS `body.exporting`)

## L3. 좌표 계산에서 화면 축소율을 빼먹음
- **What**: 블록을 드래그하면 마우스보다 2배 빠르게/엉뚱하게 이동.
- **Why**: 화면은 0.5배 축소인데 픽셀 델타를 그대로 캔버스 좌표(1080 기준)에 더함.
- **Fix**: 이동·리사이즈 시 `(deltaPx)/--view`로 역산. (섹션 3) **캔버스 좌표를 다루는 모든 곳에서 `--view` 보정을 잊지 말 것.**

## L4. 선택 서식 툴바 클릭 시 선택이 풀림
- **What**: 텍스트를 선택하고 B를 누르면 선택이 해제돼 서식이 적용 안 됨.
- **Why**: 툴바 버튼을 클릭하면 포커스가 이동하며 selection이 collapse됨.
- **Fix**: 툴바 `mousedown`에서 `preventDefault()`로 선택 유지, 적용 직전 `restore()`로 저장해둔 Range 복원. (섹션 4)

## L5. contenteditable의 인라인 폰트 크기 조절이 까다로움
- **What**: `execCommand('fontSize')`는 1~7 단계뿐이라 px 단위 조절 불가.
- **Why**: 레거시 명령의 한계.
- **Fix**: 선택 Range를 `<span style="font-size:Npx">`로 `surroundContents`(실패 시 extract+insert)하고 선택을 그 span으로 복원. (섹션 4 `stepFontSize`)

## L6. 도형 안 텍스트는 "통째 블록 삭제"와 "내부 텍스트 편집"을 구분해야 함
- **What**: 사용자가 도형 속 글자를 넣고/빼고 싶어함.
- **Why**: 도형(비교카드·체크리스트 등)은 contenteditable 컨테이너. 한 줄 추가는 Enter, 도형 자체 제거는 블록 삭제.
- **Fix**: 내부는 클릭 후 직접 타이핑(줄 추가/삭제), 도형 단위는 블록바 🗑. 두 동작이 섞이지 않게 블록바는 블록 선택 시에만 표시. (섹션 3·6)

## L7. localStorage 용량(이미지 dataURL)
- **What**: 큰 이미지를 여러 장 넣으면 저장 실패 가능.
- **Why**: 이미지를 base64 dataURL로 state에 저장 → 5MB 안팎 한도 초과 가능.
- **Fix**: `save()`를 try/catch로 감싸 실패해도 앱이 죽지 않게 함. 경고만 콘솔 출력. (섹션 1) 향후: 이미지 압축/IndexedDB 고려.

## L8. 이동(자유 배치) 시 위계가 깨지고 옮긴 텍스트가 클릭 불가 ★사용자 지적
- **What**: 블록 "위치이동"을 누르면 (1) 그 아래 텍스트가 위로 올라오고 (2) 옮기려던 텍스트가 클릭 불가가 됨.
- **Why**: ① 블록을 `.flow`(flex 중앙정렬)에서 빼서 absolute로 바꾸면 남은 블록이 즉시 재중앙정렬되어 "점프". ② absolute로 뺀 블록의 `z-index`가 `auto`라, `z-index:2`인 `.flow`(슬라이드 전체를 덮는 absolute 레이어) 아래로 깔려 클릭이 `.flow`에 가로채임.
- **Fix(v2)**: **flow/free 이원 모델을 폐기.** 모든 블록을 처음부터 absolute로 둠(최초 1회 `bakeLayout()` 측정으로 좌표 굳힘). 슬라이드 전체를 덮는 클릭 가로채기 레이어 없음. → 이동해도 다른 블록이 움직이지 않고, 겹침/클릭 문제 없음. (섹션 1·2·3)

## L9. 한번 수정하면 그 블록이 클릭 불가가 됨 ★사용자 지적
- **What**: 텍스트를 한번 편집한 뒤 다시 클릭하면 선택/편집이 안 됨.
- **Why**: 항상 `contenteditable=true`라 클릭=캐럿/선택과 이동·선택 로직이 충돌. L8의 stacking 문제와도 겹침.
- **Fix(v2)**: **클릭=선택/이동, 더블클릭=편집** 모델 도입. 비편집 상태는 `contenteditable=false` + `user-select:none`(드래그=이동), 더블클릭 시에만 편집 진입, 바깥 클릭 시 편집 종료. (섹션 3)

## L10. 모든 슬라이드를 세로로 나열 → 한 장 집중 편집이 어려움 ★사용자 지적
- **Fix(v2)**: **페이지형(한 장씩) 뷰.** 상단 페이저(‹ n/N ›)·하단 필름스트립으로 이동. `cur` 인덱스의 슬라이드만 렌더. (섹션 2·9)

## L11. 떠 있는 편집 툴바가 너무 작음 / 안내문이 난잡하고 줄바꿈이 어색 ★사용자 지적
- **Fix(v2)**: 블록 바·선택 서식 툴바 크기/패딩 확대, 라벨 추가. 안내문은 짧은 한 줄로 정리하고 토글로 접음. 도구 설명은 좌측 독 버튼 `title` 툴팁으로 이전.

## L12. 이미지 편집 UI가 슬라이드마다 인라인으로 깔려 번잡 ★사용자 지적
- **Fix(v2)**: 좌측 독의 🖼 버튼을 누르면 **옆에 뜨는 플로팅 이미지 패널**에서 현재 슬라이드 이미지의 줌·가로·세로·채움·제거를 조절. (섹션 7)

## L13. 상단 브랜드/페이지번호를 숨기거나 지울 수 없음 ★사용자 지적
- **Fix(v2)**: 브랜드·핸들(페이지번호)·진행점(dots)을 일반 **블록으로 변환**해 다른 요소처럼 이동·삭제·편집 가능. (섹션 0·1 bakeLayout)

## L14. 편집 툴바가 작게 보임 — 변환(scale)된 슬라이드 안에 둔 탓 ★사용자 지적
- **What**: 블록 선택 시 뜨는 편집 바(✥ A− A+ …)가 화면에서 너무 작게 보임.
- **Why**: 블록 바가 `transform:scale(var(--view))`(=0.6)로 축소된 `.slide` **내부**에 있어 60%로 함께 줄어듦. (반면 `position:fixed`인 선택 서식 툴바 `.fmt`는 정상 크기였음.)
- **Fix(v2.1)**: 블록 바를 **슬라이드 밖의 단일 `position:fixed` 오버레이(`#blockbar`)**로 분리. 선택/드래그/스크롤 시 선택 블록의 화면 좌표 위로 재배치(`positionBlockBar`). 축소 영향 없음. 리사이즈 핸들 `.rz`는 캔버스 안에 두되 `scale(calc(1/var(--view)))`로 역보정해 잡기 쉽게. (섹션 3)
- **규칙**: 스케일된 캔버스 안의 UI는 항상 화면 기준 크기를 갖도록 `fixed` 오버레이 또는 `1/--view` 역보정을 쓸 것.

## L15. 블록끼리의 정렬 유도선이 없음 ★사용자 지적
- **What**: 텍스트박스를 옮길 때 캔버스 고정선(중앙·안전영역)만 있고, **다른 텍스트박스와의 정렬선**이 없어 맞추기 어려움.
- **Fix(v2.1)**: 드래그 중 같은 슬라이드의 **다른 블록들의 left/center/right·top/middle/bottom**을 스냅 후보로 모아, 드래그 박스의 6개 모서리/중심이 ±11px 안에 들면 흡착하고 해당 좌표에 정렬선 표시. (섹션 3 `startDrag`/`blockGeom`) 파워포인트 스마트 가이드와 동일한 동작.
