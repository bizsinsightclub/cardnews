# 삼신노트 카드뉴스 에디터 — 디자인 스펙 (v3)

v3(템플릿형 에디터, `index.html`) 기준. 구 글래스 에디터의 스펙은 git 히스토리와 `legacy/index.html`에서 확인.

## 0. 두 개의 시각 언어 — 절대 섞지 않는다

| 층 | 역할 | 색 변수 | 변경 주체 |
|---|---|---|---|
| **에디터 셸** | 툴바·서식 바·패널·토스트·오버레이 | `--ui-*` (고정) | 아무도 (JS 변경 금지) |
| **슬라이드 캔버스** | 카드 1080×1350 + 무대 배경 | 테마 변수 (`--page` 등) | 테마 패널 → `.stage`/`.card` 인라인 |

원칙: **테마를 아무리 바꿔도 에디터 UI는 그대로여야 한다.** 과거에 두 층이 같은 변수를 공유해 "색을 바꾸면 툴바까지 변한다"는 혼란이 있었다(lesson.md L19).

## 1. 에디터 셸 토큰 (`--ui-*`, 고정)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--ui-bg` | `#efece6` | 앱 배경 |
| `--ui-surface` | `#f8f5ee` | 버튼·패널 표면 |
| `--ui-ink` | `#2e2823` | UI 텍스트, 토스트 배경 |
| `--ui-mute` | `#8a8175` | 보조 텍스트·힌트 |
| `--ui-accent` | `#a07d38` | 포인트(호버·활성·강조), primary 버튼 |
| `--ui-accent-soft` | `#cbb489` | 포인트 연한색 |
| `--ui-border` | `rgba(120,100,70,.28)` | 모든 테두리 |
| `--ui-danger` | `#b25a41` | 삭제·경고 |

- 툴바/서식 바 배경: `rgba(248,245,238,.92~.96)` + `backdrop-filter:blur(10px)`, sticky.
- 활성 상태(`.chip.act`, `.scope-btn.act`): `--ui-accent` 배경 + 흰 글자.
- 패널: `--ui-surface` 카드, radius 12–14px, `0 16~20px 40~50px rgba(120,100,70,.30)` 그림자.

## 2. 캔버스 테마 변수 (기본 = 베이지·골드)

`:root`에 기본값, JS가 `.stage`(전역 테마)·`.card`(카드별 오버라이드) 인라인 변수로 덮어씀. 테마 패널 그룹·라벨과 1:1 대응:

| 그룹 | 변수 | 라벨 | 기본값 |
|---|---|---|---|
| 배경 | `--page` | 무대(주변) 배경 | `#e3d7c0` |
| 배경 | `--ink` | 카드 배경 | `#f3ebd9` |
| 글자 | `--ivory` | 제목 글자 | `#2e2823` |
| 글자 | `--body-ink` | 본문 글자 | `#4a4138` |
| 글자 | `--mute` | 보조(흐린) 글자 | `#8e8170` |
| 글자 | `--sub-ink` | 표지 부제 글자 | `#6b5f50` |
| 포인트 | `--gold` | 포인트 색 | `#a07d38` |
| 포인트 | `--gold-soft` | 포인트 연한색 (구분선) | `#cbb489` |
| 기호 | `--red` | 빨강 (❌) | `#b25a41` |
| 기호 | `--green` | 초록 (⭕) | `#6e7a41` |

- 변수명은 저장 키(하위호환)이므로 바꾸지 말고 **라벨만** 다듬는다.
- 프리셋 6종: 베이지(기본)·소프트 핑크·세이지 민트·스카이 블루·웜 그레이·다크 — 기호색(`--red/--green`)은 프리셋이 건드리지 않는다.
- 이미지 오버레이는 카드 유효 `--ink`에서 JS가 rgba 그라데이션(0.86/0.60/0.64/0.90)을 계산해 인라인 적용.

## 4. 컴포넌트 UI

### PPT식 서식 바 (`.fmtbar`, 편집 모드에서만)
그룹 순서: **글자**(크기 입력 + 가▲▼ + B/I/U/S + 형광펜·글자색 팝오버) → **스타일**(특대~작게 6프리셋) → **문단**(왼/중/우 정렬, ≡∓ 줄간격, 자간∓) → 기타(↶↷·서식 지우기·😊). 선택할 때마다 상태(크기·B/I/U/S·정렬·색 표시줄)가 실시간 반영.

### 색 팝오버 (`.popover`)
6열 스와치 그리드(26px, hover scale 1.14) + `∅`(해제) + `<input type=color>` 커스텀. 글자색 스와치는 **현재 테마의 토큰 색**을 실시간 반영.

### 테마 패널 (`.theme-panel`)
스코프 스위치 `[🌍 전체 적용 | 🎴 이 카드만 (n쪽)]` → 프리셋 칩 → 그룹 헤딩별 색 행(컬러 피커 + HEX 입력 동기화) → 리셋(라벨이 스코프 따라 변함).

### 내 작업 (`.home-panel`, 전체 화면)
Canva 홈식 그리드(`minmax(190px,1fr)`): 4:5 썸네일, 인라인 이름 변경 input, 수정 시각, 열기/복제/HTML/삭제(2단계 확인 — `.armed` 상태 2.5초). 현재 문서는 `--ui-accent` 테두리 + "편집 중".

### 버전 패널 (`.ver-panel`)
스냅샷 목록(시각 + 라벨) + 복원/삭제 + "지금 스냅샷 만들기". 복원은 반드시 안전 백업 선행.

## 5. 모션

- 버튼/칩 hover: 0.12–0.15s ease 색 전환. 스와치 hover: scale(1.14).
- 무대 배경 테마 전환: `background .15s`.
- 토스트: 0.18s 슬라이드 업, 1.8s 후 자동 소멸.

## 6. 디자인 원칙

1. **셸은 비켜선다** — 카드가 주인공. UI는 저채도 고정 팔레트.
2. **WYSIWYG** — 화면 카드 = 저장 PNG (편집 표식은 캡처 시 제거, L1/L2).
3. **그림판만큼 쉬운 색 고르기** — 스와치 우선, HEX는 보조.
4. **원본 비주얼 불변** — 편집 기능을 더하되 슬라이드 타이포·컴포넌트 값은 건드리지 않는다.
5. **이름은 눈에 보이는 대로** — 색 역할명은 "제목 글자/카드 배경"처럼 위치·역할로 부른다 (내부 변수명 노출 금지).
