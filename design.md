# design.md — 삼신노트 카드뉴스 에디터 디자인 명세

이 에디터의 시각 언어는 **두 층**으로 나뉩니다.

1. **에디터 셸(Editor Shell)** — 글래스모피즘 웹 에디터 홈페이지. 도구·툴바·팔레트.
2. **슬라이드 캔버스(Slide Canvas)** — 실제 결과물인 카드뉴스. 원본 "삼신노트 퍼고베리스" 비주얼을 그대로 보존.

이 둘은 의도적으로 다른 디자인 언어를 씁니다. 셸은 반투명·블러·부유감, 캔버스는 불투명·인쇄지향·고대비.

---

## 1. 컬러 토큰

```
/* 브랜드 (슬라이드 캔버스) */
--cream  #FFF9D2   연한 크림 (배경 상단)
--peach  #FFEBCC   복숭아 (배경 하단, eq 박스)
--sky1   #BFDDF0   연하늘 (배지·강조 카드)
--sky2   #8CC0EB   하늘 (버튼·하이라이트·정렬선)
--ink    #2A3340   본문 진한 잉크
--ink2   #46505E   보조 본문
--mut    #7C8794   흐린 텍스트(핸들·주석)
--acc    #3E6B8C   진한 스카이 (소형 텍스트 가독·강조)

/* 에디터 셸 (글래스) */
--glass      rgba(255,255,255,.55)   유리 표면
--glass-brd  rgba(255,255,255,.7)    유리 테두리(하이라이트)
--glass-sh   0 8px 32px rgba(31,52,74,.18)   유리 그림자
```

배경(셸)은 3겹 radial-gradient + 대각 linear-gradient를 `background-attachment:fixed`로 깔아 스크롤해도 은은한 오로라가 유지됩니다.

## 2. 글래스모피즘 스펙

| 요소 | blur | 배경 | 테두리 | 그림자 |
|---|---|---|---|---|
| 상단바 `.topbar` | 20px saturate(150%) | rgba(255,255,255,.45) | 하단 1px rgba(255,255,255,.6) | — |
| 카드/힌트 `.glass` | 18px saturate(160%) | --glass | 1px --glass-brd | --glass-sh |
| 선택 서식 툴바 `.fmt` | 10px | rgba(40,55,72,.95) 다크 | — | 0 10px 30px /.32 |
| 블록 바 `.block-bar` | 8px | rgba(40,55,72,.92) 다크 | — | 0 6px 18px /.28 |
| 팔레트/도형메뉴 | 16px | rgba(255,255,255,.92) | 1px /.8 | 0 16px 44px /.3 |

원칙: **떠 있는 컨트롤은 다크 글래스(잉크), 패널은 라이트 글래스(화이트).** 버튼 hover는 `translateY(-1px)` + 그림자 확대로 부유감.

## 4. 레이아웃 (v2 — 한 장씩 / 좌측 독)

화면 구성:

```
┌ topbar (로고 · 사용법 · 전체 PNG) ─────────────────────┐
│ [dock]   pager (‹ n/N › 파일명)                        │
│  T 텍스트   ┌──────────────────────────┐               │
│  ◇ 도형     │                          │               │
│  🖼 이미지   │     슬라이드 1장 (현재)    │               │
│  🎨 배경     │   1080×1350 · scale 0.6  │               │
│  〰 가이드   │                          │               │
│  ─          └──────────────────────────┘               │
│  ＋⧉🗑       filmstrip [1★][2][3]…[8]                   │
│  📥↺                                                    │
└────────────────────────────────────────────────────────┘
```

- 캔버스: **1080 × 1350px**, 화면에선 `--view: 0.6`(648×810)로 축소. 좌표 계산은 항상 `/VIEW()`로 역산.
- **한 번에 한 슬라이드만 렌더**(`cur` 인덱스). 페이저·필름스트립으로 전환.
- 안전 영역: inset 62px 점선 + 절취 마크 `.cm`. 저장 시 `body.exporting`으로 숨김.
- **블록은 전부 절대좌표.** 브랜드·핸들·진행점(dots)도 블록이라 이동·삭제 가능. 최초 1회 `bakeLayout()`이 원본 레이아웃을 측정해 좌표를 굳힌다.
- `.stagewrap`은 `overflow:visible`(툴바·핸들이 캔버스 밖으로 나와도 보이게), 카드의 둥근 모서리·그림자는 `.bg`/`.overlay`가 담당.

### 인터랙션 모델
- **클릭=선택/이동**(비편집 시 `user-select:none`, 드래그로 이동), **더블클릭=편집**(`contenteditable=true`), 바깥 클릭=편집 종료. 이 분리가 "수정 후 클릭 불가" 버그를 막는다.
- 드래그 중 snap: 중앙(540)·좌안전(62)·우안전(1018)·상하 안전(62/1288)에서 ±14px 흡착, 정렬선 표시.

### 좌측 도구 독 (포토샵식)
세로 글래스 바, 아이콘+라벨 버튼, hover 시 우측 툴팁(`data-tip`). 그룹: ①편집(T·◇·🖼·🎨·〰) ②슬라이드(＋·⧉·🗑) ③출력(📥·↺). 활성 도구는 스카이 그라데이션 강조.

## 5. 컴포넌트 UI

### 선택 서식 툴바 `.fmt`
- 트리거: 텍스트 드래그 선택(`selectionchange`, 비-collapsed, `.ed` 내부).
- 위치: 선택 영역 `getBoundingClientRect()` 중앙 상단 12px, 화면 밖이면 하단으로 플립. 아래쪽 삼각 포인터.
- 버튼: **B / I / U / 🖍︎(테마 하이라이트) / 🎨(색선택 하이라이트) / 글자색 / A− / A+(글자크기) / AV−·AV+(자간) / ≡−·≡+(줄간격) / ⌫(서식제거)**. 활성 상태는 `.act`(하늘 배경).
- 인라인 vs 박스: B·I·U·색·글자크기는 **선택 구간(인라인)**, 자간·줄간격은 **텍스트박스 전체(블록)**에 적용(줄간격은 인라인으로 불가). 자간·줄간격은 `adjustBoxStyle()`이 `b.letterSpacing`(em)·`b.lineHeight`(비율)를 갱신.
- 클릭 시 선택 유지를 위해 `mousedown`에서 `preventDefault`.

### 블록 바 `#blockbar`
- **`position:fixed` 단일 오버레이**(슬라이드 밖). 스케일된 캔버스 안에 두면 60%로 작아지므로(L14) 화면 좌표로 띄우고 선택/드래그/스크롤마다 `positionBlockBar()`로 재배치.
- 트리거: 블록 선택 시 그 블록 위에 표시.
- 버튼: **✥(이동 손잡이) / A− / A+ / 🎨(블록 배경색) / ⧉(복제) / 🗑(삭제)**.
- 선택 블록은 `:after`로 하늘 점선 외곽선 + 우측 너비 리사이즈 핸들 `.rz`(캔버스 안, `1/--view` 역보정).

### 컬러 팔레트 `.palette` (그림판식)
- 10×N 스와치 그리드(40색: 그레이스케일/웜/쿨/브랜드/포인트) → `<input type=color>` → hex 텍스트 입력.
- `allowNone`이면 "기본값으로 되돌리기" 링크.
- 트리거 버튼 기준으로 위치 잡고, 화면 경계 보정.

### 도형 추가 메뉴 `.addmenu`
- 9종: 제목·본문·배지·비율배지·강조박스·비교카드2단·체크리스트·주의문구·브랜드라인.
- 추가 시 선택 슬라이드 중앙(x140,y520,w760)에 `free` 블록으로 삽입.

## 6. 모션
- 버튼 hover 0.12s ease (y-1px + 그림자), 스와치 hover scale(1.18).
- 드래그 중 snap 정렬선(`.snapline`): 캔버스 가이드(중앙 540·675, 안전 62/1018/1288)에 더해 **같은 슬라이드 다른 블록들의 left/center/right·top/middle/bottom**을 후보로 모아, 드래그 박스의 6개 모서리/중심이 ±11px 안에 들면 흡착(파워포인트식 스마트 가이드).

## 7. 디자인 원칙
1. **셸은 비켜선다.** 결과물(캔버스)이 주인공. 떠 있는 컨트롤은 필요할 때만 나타나고 캡처에서 제거된다.
2. **WYSIWYG.** 화면의 카드 = 저장될 PNG. 가이드·핸들·외곽선만 예외(`body.exporting`에서 숨김).
3. **그림판처럼 단순한 색 선택.** 전문 컬러휠 대신 즉시 고르는 스와치 + hex.
4. **원본 비주얼 불변.** 편집 기능을 얹되 슬라이드 디자인 토큰/크기는 바꾸지 않는다.
